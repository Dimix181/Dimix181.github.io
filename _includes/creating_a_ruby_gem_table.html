Folder Description
bin This folder is responsible for the starting your gem code. It can be called from your CLI.
app_name This file is equivalent to your enviroment file. It holds all the required path of files that your application needs inorder to function properly
spec Code for testing your gem
app_name_spec.rb Code for testing your gem
spec_helper.rb Used for testing your code
.gitignore This file determine what to include when committing and pushing your code out to your repository
.rspec Code for testing your code in a test driven development environment.
.travis.yml Help ease the release of your gem once its completed and ready for deployment.
app_name.gemspec Contains your gem's specifications
CODE_OF_CONDUCT.md Rules for contributing
Gemfile This file is responsible for loading your projects gem dependencies. More detail information can be found at bundler website https://bundler.io/v1.5/gemfile.html
LICENSE.txt The rights for your gem
Rakefile Similar to Gemfile. Allows your to specify dependencies and ruby executables
README.md Description of what your gem does
